# Learn Pegium

This section is the recommended learning path for Pegium.

It is organized around the same idea as the Langium documentation: start with a
practical workflow, then follow the steps in order until you have a complete
language.

## Start here if

- you are building your first Pegium language
- you want the shortest path from clone to working language services
- you prefer a guided sequence over jumping between API pages

## Where to start

### Workflow

The fastest way to understand Pegium is the
[workflow section](workflow/index.md). It explains the recommended sequence
from repository build to grammar, AST, references, validations, and editor
services.

### Examples

If you prefer learning from a concrete codebase, keep the
[examples overview](../examples/index.md) open while reading the workflow.

### Deeper topics

Once you know the common workflow, use [Recipes](../recipes/index.md) for
targeted customization and [Reference](../reference/index.md) for precise
concepts.

## Suggested reading rhythm

Read one workflow step, compare it with the closest shipped example, then come
back to the next step. That loop tends to work better than trying to absorb
all of `Reference` first.
