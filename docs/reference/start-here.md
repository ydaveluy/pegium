# Start Here

Use this page when you know you need the reference section, but you are not yet
sure which page contains the right level of detail.

## Read Glossary first if

- a Pegium term appears in another page and you want the short definition
- you want a compact refresher on AST, CST, references, scopes, and services

Go to [Glossary](glossary.md).

## Read Configuration Services first if

- you want to understand how a language is wired together
- you are looking for `makeDefaultServices(...)`
- you want the baseline service composition model

Go to [Configuration Services](configuration-services.md).

## Read Semantic Model first if

- you are shaping your AST types
- you want to understand how parser assignments define the language model
- you need the AST/CST/reference mental model

Go to [Semantic Model](semantic-model.md).

## Read Document Lifecycle first if

- you are working on indexing, linking, validation, or LSP behavior
- you want to know when data becomes available during a build
- you are reasoning about caching or invalidation

Go to [Document Lifecycle](document-lifecycle.md).

## Read Grammar Reference first if

- you need the exact parser DSL surface
- you are checking rule, terminal, skipper, or infix syntax
- you want canonical snippets rather than a tutorial

Go to [Grammar Reference](grammar-reference.md).

## Read the API-oriented pages next if

- you already know the concept and now need extension points
- you are implementing formatter, completion, or workspace customization

Good next pages:

- [Services and Modules](services-and-modules.md)
- [Formatter DSL](formatter-dsl.md)
- [Default LSP Services](lsp-services.md)
- [Completion Provider](completion-provider.md)
- [Workspace Concepts](workspace.md)

## Rule of thumb

Use `Learn` to discover the order of implementation, `Recipes` to solve a
targeted problem, and `Reference` to confirm the exact concept or API.
