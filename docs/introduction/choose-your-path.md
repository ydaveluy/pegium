# Choose Your Path

Use this page when you know you want to work with Pegium, but you are not yet
sure which section of the documentation matches your goal.

## Start with Introduction if

- you are still deciding whether Pegium fits your project
- you want the high-level picture before reading APIs
- you are coming from Langium and want the concept mapping first

Useful pages:

- [What is Pegium?](index.md)
- [Why Pegium](why-pegium.md)
- [Pegium for Langium Users](langium-users.md)

## Start with Learn if

- you want the recommended end-to-end workflow
- you are building your first language with Pegium
- you prefer a linear path over jumping between API pages

Useful pages:

- [Learn Pegium](../learn/index.md)
- [Workflow](../learn/workflow/index.md)

## Start with Recipes if

- you already know which subsystem you want to customize
- your question sounds like "How do I...?"
- you want examples for scoping, validation, caching, or multiple languages

Useful pages:

- [Recipes](../recipes/index.md)
- [Qualified Names](../recipes/scoping/qualified-names.md)
- [Dependency Loops](../recipes/validation/dependency-loops.md)
- [Caches](../recipes/performance/caches.md)

## Start with Reference if

- you want exact terminology or API-oriented explanations
- you need to understand how services, documents, or the semantic model fit
  together
- you already know the workflow and want canonical subsystem descriptions

Useful pages:

- [Reference](../reference/index.md)
- [Start Here](../reference/start-here.md)
- [Glossary](../reference/glossary.md)

## Start with Examples if

- you learn best from working code
- you want a template close to your target language
- you prefer to inspect a complete implementation before reading abstractions

Useful pages:

- [Examples Overview](../examples/index.md)
- [Arithmetics](../examples/arithmetics.md)
- [DomainModel](../examples/domainmodel.md)
- [Requirements](../examples/requirements.md)

## Rule of thumb

Read `Introduction` to orient yourself, `Learn` to build something, `Recipes`
to solve targeted tasks, and `Reference` to confirm exact concepts and APIs.
